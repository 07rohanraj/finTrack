# Spending Dashboard

Interactive web dashboard for analyzing PhonePe transaction data. Built with Python (backend) and Chart.js (frontend).

## Quick Start

```bash
cd spending-app
python server.py
```

Open **http://localhost:8420** in your browser.

## Requirements

- Python 3.8+
- pandas
- openpyxl

No other dependencies needed. The server uses Python's built-in HTTP module.

## Project Structure

```
spending-app/
  server.py          # Backend: reads Excel, serves API + static files
  static/
    index.html       # Page structure
    style.css        # Dark theme styles
    app.js           # Charts, filtering, table logic
```

## Features

- **Filters** — Category, date range, weekday/weekend, payee search
- **Charts** — Monthly trend, category breakdown, top payees, hourly patterns, quarterly, day of week
- **Table** — Sortable, paginated (50 rows), searchable transaction list
- **Summary cards** — Total spent, monthly average, median, payees, biggest transaction

## How It Works

1. `server.py` reads `TransactionStatement.xlsx` from the parent directory on startup
2. Categorizes all debit transactions (excludes personal transfers, investments, bank transfers)
3. Serves the full dataset as a single JSON API call (`/api/data`)
4. Frontend fetches data once, caches it, and does all filtering/rendering client-side
5. Charts update instantly on filter changes — no server roundtrips needed

## Data

Expects `TransactionStatement.xlsx` in `D:\Learning\fin\` (parent of this folder). The Excel file is generated by running `parse_transactions.py` from the parent directory.
